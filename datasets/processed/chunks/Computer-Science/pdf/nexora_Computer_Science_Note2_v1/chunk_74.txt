SRTF vs SJF Equivalence Statement: Shortest Remaining Time First (SRTF) behaves like Shortest Job First (SJF) if the. set of processes is fixed and all processes are CPU bound. Answer:. True. Justification:. When there are no new arrivals and all processes are known in advance, SRTF always picks. the process with the least remaining burst time. Since no new process arrives, the least. remaining time is simply the original burst time. SJF also picks the process with the shortest burst time next. Thus, both schedules will be identical in order and context switches for fixed, CPU-bound jobs. 2. CPU Bound vs. I/O Bound Processes . I/O Bound processes spend most time doing input/output; require little CPU. Context Switches: I/O bound processes cause more frequent context switches, because they quickly relinquish CPU to wait for I/O, while CPU bound processes retain CPU longer, causing fewer context switches. 3. Task sets with total utilization > 69.3% (Liu & Layland bound) cannot be guaranteed to be schedulable by Rate Monotonic, but EDF can schedule any set as long as total CPU utilization 100%.
