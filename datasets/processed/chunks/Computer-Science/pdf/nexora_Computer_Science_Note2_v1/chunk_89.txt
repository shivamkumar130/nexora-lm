Q1 - equal-difference property I AES without SubBytes (5+5+5) Definition (equal-difference property). A function (f:{0,1}^N\to{0,1}^N) has the equal-difference property if x_1\oplus x_2 = x_3\oplus x_4 \quad\Longrightarrow\quad f(x_1)\oplus f(x_2) = f(x_3)\oplus f(x_4) for all (x_1,x_2,x_3,x_4). f(x_1)\oplus f(x_2)=(\alpha x_1+\beta)+(\alpha x_2+\beta)=\alpha(x_1+x_2) because (\beta+\beta=0) in characteristic 2. If (x_1\oplus x_2=x_3\oplus x_4) then (\alpha(x_1+x_2)=\alpha(x_3+x_4)) and thus (f(x_1)\oplus f(x_2)=f(x_3)\oplus f(x_4)). So affine maps over GF((2^k)) have the property (b) AES round operations: SubBytes is nonlinear; ShiftRows, MixColumns and AddRoundKey are linear (or affine) over the vector space ((\mathrm{GF}(2))^{128}). If you remove SubBytes every round is affine, so the whole block encryption becomes an affine map. E(x)=A x + b for some linear map (A) and vector (b). For such an affine map E(x_1)\opIus E(x_2) = A(x_1\opIus x_2) (c) Why this is broken / how to decrypt quickly E(x_1)\opIus E(x_2)=A(x_1\opIus x_2) Thus an adversary who can obtain encryptions of chosen plaintexts (a chosen-plaintext attack) can recover the linear map (A) and vector (b) with a small number of encryptions: Request (E(0)=b) For each basis vector (e_i) (128 vectors) request (E(e_i)). Then A e_i = E(e_i)\oplus b, so the columns of (A) are obtained. (Equivalently one can use difference queries (E(x)\oplus E(x\oplus e_i)).) Once (A) and (b) are known we invert (A) (Gaussian elimination) and compute for any ciphertext (y) the plaintext x = A^{-1}(y\oplus b). Cost: about (128+1) chosen encryptions and polynomial time linear algebra - vastly faster than brute force (2^{128}) (If attacker only has a handful of known plaintexts but can choose plaintexts, the same idea with differences gives the same result. The main point: affine encryption is totally insecure against chosen-plaintext recovery.) Q2 - Vigenere mod 2 on alphabet ({a,b}).
