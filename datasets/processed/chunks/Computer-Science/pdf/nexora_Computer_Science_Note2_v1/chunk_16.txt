Hidden Terminal: Two nodes can't sense each other; collision occurs at receiver. Exposed Terminal: Node refrains from sending even if it wouldn't interfere Solution: MACA / MACAW protocols using RTS/CTS and ACKs. Module 2: Flow Control and ARQ Protocols. 2.1 Flow Control. Coordinates the sender and receiver speeds. Prevents buffer overflow at the receiver. Mechanism: Sender can send a certain number of frames before waiting for ACKs 2.2 Stop-and-Wait Protocol. Send one frame at a time, wait for ACK. Problems: Lost frame, lost ACK, delayed frame/ACK.. 2.3 Stop-and-Wait ARQ Adds timeout and retransmission. 2.4 Sliding Window Protocol Sends multiple frames before receiving ACK. Window Size (N): Number of frames in transmission without ACK. Go-Back-N ARQ: Retransmit all frames in window if one frame lost. Selective Repeat ARQ: Retransmit only lost frames.. Example Problem: Frames: 11, Window Size: 4, Lost Frame: 5th. Solution Steps: 1. Send frames 1-4 -> 4 frames sent 2. Send 5th frame -> 5 frames sent 3. Send 6th frame after ACK 1 > 6 frames sent 4.
