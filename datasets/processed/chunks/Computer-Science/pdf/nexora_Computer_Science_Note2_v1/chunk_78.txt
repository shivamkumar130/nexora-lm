If yes how? If no, how to deal with starvation? Answer: Yes - detection and mitigation both possible. Detection: The system can monitor per-process waiting times (time spent ready but not scheduled) or number of times preempted. If a process's waiting time exceeds a configured threshold, the system can treat that as starvation. Mitigation (common technigues): Aging: gradually increase the priority of processes that wait long, so they eventually get CPU. Fair schedulers: use fair-share / proportional-share schedulers (CFS in Linux style) which bound starvation.. Admission / resource limits: limit how many high-priority tasks a user or class can have so low-priority tasks aren't starved. So detection is practical; the usual response is to apply aging or use a scheduler that guarantees progress.
