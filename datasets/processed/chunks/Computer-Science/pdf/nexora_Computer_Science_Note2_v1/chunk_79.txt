Code (interpreted from image): pid_t pid; pid = fork(); if (pid == O) { /* child process */ fork(); thread_create(...); fork(); (a) How many unique processes are created?. Answer: 6 processes total. Reason: Start with 1 -> first fork() -> 2 processes (parent + child). In the child, the inner fork() creates a third process (child's child). At that point there are 3 processes. After exiting. the if block, the final fork() is executed by all current processes (the original parent and both. child processes), doubling them from 3 -> 6.. (b) How many unique threads are created? Answer: 2 new threads (created by thread_create), assuming thread_create is executed in both the child and the child's child (both execute the code after the inner fork). So two threads are created; the total number of threads depends on whether the forks duplicate threads in a particular environment, but the direct thread_create invocations run twice -> two threads. (If the question asks "how many threads exist at the end?" you must also count the main. threads in each process - but typical exam wants "how many new threads created by. thread_create()" = 2.) Q8 To what degree do the following algorithms favor CPU-bound processes? (brief) Rate them qualitatively: (favored / neutral / disfavored) (a) FCFS (First-Come, First-Serve) - Neutral / can favor long CPU-bound jobs if they. arrive first. FCFS can let a long CPU job hold CPU causing I/O-bound ones to wait (convoy effect) (b) SJF (Shortest Job First) - Disfavors CPU-bound (favours short jobs). SJF prefers short CPU bursts (l/O-bound), so CPU-bound (long jobs) wait longer. (c) SRtF (Shortest Remaining Time First) - Disfavors CPU-bound (same as SJF behavior but preemptive).
