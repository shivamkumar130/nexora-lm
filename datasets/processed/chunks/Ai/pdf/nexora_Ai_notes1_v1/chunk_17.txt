Credit goes to Sutton and Barto. 18 Solution - Lights-Out (4x4) - parts (A)-(D) Below I give a clean, course-ready answer for each part. I also use the exact initial board from and (row 3, col 4) (rows/cols counted from top-left = (1,1)). I'll mark 1 = ON, 0 = OFF. Initial state (4x4, row-major): Row\Col 1 2 3 4 1000 0000 0001 0000 1) (A) - Pose as a state-space search problem State representation: a 4x4 binary matrix (or a 16-bit vector) s where s[i, j] e {0, 1} (0 = OFF, 1 = ON). Total state space size = (2^{16}) Initial state: as above. Actions: click(r,c) for any cell (r,c) with 1 r,c 4.Applying click(r,c) toggles the state (0<>1) of the cell (r, c) and its orthogonal neighbours (r1, c) and (r, c1) (only those inside the board). Transition function: Result(s, click(r, c)) = new state with those 1-5 bits toggled. Goal test: all cells OFF (all zeros). Cost function: unit cost per click (cost = 1). (You can also consider uniform-cost if clicks had differing costs.) Search space properties: branching factor b = 16. Depth = number of clicks. Note: because. clicking twice on same cell cancels (mod 2), sequences can be considered modulo 2 there are linear dependencies (useful later).. 2) (B) Suggested heuristic(s) We want an admissible (never-overestimates) heuristic estimating the minimum number of clicks to reach all-off. Two simple heuristics: h0(s) = 0 - trivial admissible heuristic. h1(s) = ceil((#ON(s)) / 5) Reason: a single click affects at most 5 cells (center cell + 4 neighbours).
