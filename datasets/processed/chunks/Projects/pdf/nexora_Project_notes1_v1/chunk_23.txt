"Object Detection Using YOLOv8" (2024) *This presentation synthesizes key findings from multiple research papers on YOLOv8 architecture and performance.* Operating Systems (Cs303) Project Report Title: Latency Monitoring via PMU Counters Group No: 16 Members: .Dev Saraswat (202351173) Shivam Kumar (202351130) .Satish Chauhan (202352330) Course Instructor: Dr. Bhupendra Kumar T.A.: Rishabh Sir Date of Submission: September 2025 Abstract Monitoring latency in computer systems is critical for ensuring performance and user experience. Tail latency spikes, especially at the 95th, 99th, and 99.9th percentiles, can degrade application responsiveness. This project leverages Performance Monitoring Unit (Pmu) counters to correlate hardware-level metrics with latency behavior. Using the Linux perf tool and perf_event_open syscall, we collected and analyzed CPU performance events. Results indicate strong correlations between branch mispredictions, CPI, IPC, bus cycles, and latency. Introduction Modern systems demand consistent performance, but latency spikes are common under high Ioad. Identifying their root cause requires observability at the hardware level. PMU counters provide low-level metrics (cycles, instructions, cache misses, bus cycles, branch mispredictions) which can reveal inefficiencies. Our goal was to monitor PMU counters in real time, correlate them with workload latency, and visualize the impact of microarchitectural behavior on system performance. Problem Statement: How can PmU counters be used to detect and analyze latency spikes in modern processors? Objective: Capture PMU counters programmatically. Correlate with latency traces. Identify counters linked with p95/p99/p999 latency. Prototype a lightweight monitoring framework..
