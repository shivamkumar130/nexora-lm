Counter Selection We tracked the following PMU counters: cpu_atom/cycles, cpu_core/cycles -> measure execution cycles.. cpu_atom/instructions, cpu_core/instructions -> instruction throughput. cpu_atom/cache-references, cpu_core/cache-references -> memory hierarchy behavior. cpu_atom/cache-misses, cpu_core/cache-misses -> cache pressure and latency contributors. cpu_atom/branch-instructions, cpu_core/branch-instructions -> control flow operations. cpu_atom/branch-misses, cpu_core/branch-misses -> misprediction penalties and pipeline hazards. interconnect contention. timestamp -> align hardware counter data with workload traces.. Selection Justification: Cycles & Instructions -> for CPI (Cycles per Instruction) and IPC (Instructions per Cycle). Cache References & Cache Misses -> to evaluate memory performance and cache behavior. Branch Instructions & Branch Misses -> to track control hazards and stalls. Bus Cycles -> to measure memory bandwidth usage. Timestamp -> for correlation with latency events..
