Methodology: The proiect followed the CRISP-DM (Cross-Industry Standard Process for Data Mining) methodology: Business Understanding: Defined the problem and project objectives Data Understanding: Explored and visualized the medical data Data Preparation: Cleaned and transformed the data Modeling: Developed and trained machine learning models o Evaluation: Assessed model performance. o Deployment: Planned for clinical applications For modeling, we experimented with three different algorithms: o Random Forest Classifier. o XGBoost Classifier o Logistic Regression The models were evaluated based on accuracy, precision, recall, F1-score, and AUC-ROC score. 5. Data Preprocessing: The following preprocessing steps were applied to the data: Date Conversion: Converted diagnosis and treatment dates to datetime format Treatment Duration Calculation: Calculated treatment duration in days o Handling Missing Values: Checked for and handled any missing values Feature Scaling: Applied standardization to numerical features o Categorical Encoding: Converted categorical variables to numerical representations o Train-Test Split: Divided data into training (70%) and testing (30%) sets Class Imbalance Handling: Applied SMOTE to address class imbalance that the models would not be biased by the scale of different features 6. Feature Engineering: Several new features were created to improve model performance: o AGE GROUP: Categorical age groups o BMI_CATEGORY: BMI categories (underweight, normal, overweight obese) TREATMENT INTENSITY: Composite score based on treatment type and duration COMORBIDITY_COUNT: Count of comorbidities (hypertension, asthma, cirrhosis) RISK SCORE: Composite risk score based on multiple factors age-related risk factors, treatment intensity, and comorbidity burden.
