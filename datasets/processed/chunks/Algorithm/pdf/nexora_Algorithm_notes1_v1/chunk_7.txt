Condition to store all (m) patterns using Hebb rule (no interference): For the multi-pattern case (W=\sum_{k=1}^m x^{(k)}{x^{(k)}}^T - ml) we can compute x^{(p)T}W x^{(p)} = n^2 - mn +\sum_{k\ne p} (x^{(p)T}x^{(k)})^2. If the stored patterns are mutually orthogonal (i.e. (x^{(p)T}x^{(k)}=0) for (k\ne p)), then the. cross terms vanish and each stored pattern yields the same maximal value, so each (x^{(p)}) is an energy minimum. Thus the usual sufficient condition: patterns must be (approximately) orthogonal / uncorrelated so Hebbian learning stores them as separate minima (no crosstalk). In practice exact orthogonality is rare; with many patterns cross-talk arises and retrieval fails if (m) is too large (classical capacity ~0.14(n) for random uncorrelated patterns).
