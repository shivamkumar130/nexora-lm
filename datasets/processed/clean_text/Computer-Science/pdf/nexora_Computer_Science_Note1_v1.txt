Resistive Respiration Monitoring System
1. Overview
The proposed system is a wearable resistive respiration monitoring device that measures.
breathing-induced resistance variations using a constant-current excitation method. The sensor
signal is conditioned using analog circuitry and digitized by an ESP32 microcontroller for further
processing and wireless transmission.
2. System Block Architecture
The system consists of four main functional blocks:.
1. Resistive Respiration Sensor Interface
2. Analog Front-End (Signal Conditioning)
3. Microcontroller Unit (MCU)
4. Power Management Unit
3. Resistive Respiration Sensor Interface.
A two-pin connector (J_SENsOR1) is used to interface the external resistive respiration
sensor.
SENsOR+: Connected to the excitation and measurement path.
SENsOR-: Connected to system ground
Breathing causes small resistance changes in the sensor, which are converted into voltage.
variations by the analog front-end.
4. Analog Front-End (AFE)
4.1 Constant Current Source (U2A)
Op-Amp: TLV9062 (Unit A)
Function: Generates a stable constant current through the resistive sensor
.R_SET (33 kQ): Sets the excitation current value
This approach ensures that sensor resistance variations are linearly converted into voltage
changes.
4.2 Signal Amplifier (U2B)
Op-Amp: TLV9062 (Unit B)
Function: Amplifies the small voltage variations from the sensor
R_G (10 kQ) & R_F (100 kQ): Configure the amplifier gain
This stage improves signal amplitude while maintaining low noise performance
4.3 ADC Input Filtering
Before feeding the signal to the MCU ADC:
:R1 (100 Q): Series resistor
:C2 (100 nF): Shunt capacitor to ground
Together, they form a low-pass Rc filter that suppresses high-frequency noise and stabilizes
ADC sampling.
Output net: ADC IN
5. Microcontroller Unit (MCU)
5.1 ESP32-C3-WROOM-02
The ESP32-C3 serves as the central controller and provides:
12-bit ADC for analog signal acquisition
Digital processing and filtering
Wireless communication (Bluetooth / Wi-Fi).
5.2 Supporting Components
.C1 (0.1 F): Local decoupling capacitor
R3 (10 kQ): EN pin pull-up resistor
The conditioned analog signal is sampled through the ADc_IN pin..
6. Power Management Unit
6.1 Power Input
J2: 5V input from USB or external supply.
6.2 Voltage Regulation
U3 (AMS1117-3.3): Linear regulator providing 3.3V.
C4, C5 (10 F): Bulk capacitors for stability
.C6, C7 (0.1 F): High-frequency decoupling capacitors.
This ensures a stable and low-noise supply for both analog and digital circuits.
7. Grounding Strategy
A common ground (GND1) is used across analog, digital, and power sections.
Short ground paths and local decoupling minimize noise coupling into the analog
front-end.
8. Overall Signal Flow
Respiration -> Sensor Resistance Change
> Constant Current Excitation
> Voltage Conversion
> Signal Amplification
-> RC Filtering
> ESP32 ADC
> Digital Processing & Transmission

System software is a collection of computer programs that control a computer's hardware
computer's hardware and the user, allowing users to interact with the computer's software and
hardware. [1, 2, 3]
System software includes: [1]
Operating system (Os): The most well-known example of system software, the OS
manages all other programs on the computer. It allocates and organizes resources based
on priority, and tracks all processes occurring in the system. [1, 4]
Device drivers: A component of system software. [5]
Utility software: Programs that help with the management and maintenance of the
computer system. Examples include anti-virus, disk management tools, and data backup
tools. [6]
Basic input/output system (BlOs): A component of system software. [5]
Boot program: A component of system software. [5]
System software is different from application software, which runs on top of system software.
and directs the computer to execute user commands. System software is also coded differently.
than application software, generally using system programming languages. [5]
Some examples of system software types include: Windows, Linux, MacOS X, and BSD. [2]

Complexity Comparison:
The Parallel BOGO Sort algorithm has factorial time complexity O(n!),and
In contrast, the Parallel Sections (Sum, Max, Min) and Parallel Odd-Even

Conclusion:

which makes it computationally expensive and non-scalable. AIthough
parallelization reduces execution time by dividing work among threads
the overall complexity class remains unchanged.
In contrast, the Parallel Sections (Sum, Max, Min) and Parallel Odd-Even
Sum problems have linear time complexity O(N). These algorithms scale
efficiently with input size and benefit significantly from parallel execution
