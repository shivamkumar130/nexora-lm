Statement: If the last round of an SPN has a permutation layer, prove it does not increase the.
strength of the cipher.
Answer / Proof (informal, standard):
A permutation layer is a fixed linear reordering of bit/byte positions (an invertible linear.
relabeling of ciphertext bits (an adversary can simply apply the inverse permutation to any
ciphertext) or (b) into the representation of the final round key (we can redefine the final round
key bytes so that the permutation is already applied). Thus the cipher with the final permutation
is equivalent (from a security point of view) to the cipher without it: the mapping from
plaintext-and-key to ciphertext is only changed by a known permutation of output coordinates, or
by rewriting the key schedule - neither increases nonlinearity or key-space complexity.
Therefore the final permutation layer does not strengthen the cipher.
(Conclude: the last-round permutation is cosmetic - it can be removed or absorbed without
reducing the set of possible ciphertexts produced for given keys.)

CSMA/CD: Collision Detection used in Ethernet. Stop transmitting if collision
occurs.
CSMA/CA: Collision Avoidance used in Wireless (WiFi IEEE 802.11). Uses
RTS/CTS to avoid hidden and exposed terminal problems..
1.3 Hidden & Exposed Terminal Problems.
Hidden Terminal: Two nodes can't sense each other; collision occurs at receiver.
Exposed Terminal: Node refrains from sending even if it wouldn't interfere
Solution: MACA / MACAW protocols using RTS/CTS and ACKs.
Module 2: Flow Control and ARQ Protocols.
2.1 Flow Control.
Coordinates the sender and receiver speeds.
Prevents buffer overflow at the receiver.
Mechanism: Sender can send a certain number of frames before waiting for ACKs
2.2 Stop-and-Wait Protocol.
Send one frame at a time, wait for ACK.
Problems: Lost frame, lost ACK, delayed frame/ACK..
2.3 Stop-and-Wait ARQ
Adds timeout and retransmission.
2.4 Sliding Window Protocol
Sends multiple frames before receiving ACK.
Window Size (N): Number of frames in transmission without ACK.
Go-Back-N ARQ: Retransmit all frames in window if one frame lost.
Selective Repeat ARQ: Retransmit only lost frames..
Example Problem:
Frames: 11, Window Size: 4, Lost Frame: 5th.
Solution Steps:
1. Send frames 1-4 -> 4 frames sent
2. Send 5th frame -> 5 frames sent
3. Send 6th frame after ACK 1 > 6 frames sent
4. Send 7th frame after ACK 2 -> 7 frames sent
5. Send 8th frame after ACK 3 > 8 frames sent
6. Lost 5th frame -> retransmit window (5,6,7,8) -> 12 frames sent
7. Send remaining frames -> total frames transmitted = 13
2.5 ARQ Transmission Time Example
Given: 2,000,000 bits, frame size 1,000 bits, RTT = 50 ms, sequence number 4 bits
Stop-and-Wait ARQ: 2000 frames 50 ms = 100 s
Go-Back-N ARQ: Window size 16 -> (2000/16) x 50 ms = 6.25 s
Selective Repeat ARQ: Same as Go-Back-N if all frames arrive correctly -> 6.25 s
2.6 Optimal Window Size for Minimum Delay.
Formula: N = 1 + 2a, where a = Tp/Tt (propagation time / transmission time)
If transmission time negligible -> a -> -> N ->
Go-Back-N: Sender window N = 200o, Receiver window = 1
Selective Repeat: Sender & Receiver windows N = 2000

_ Detailed DBmS Theory Notes.
(33 DBMS Layers
1. External Layer
It is the user view of the database..
Different users see different parts of the database..
Ensures data security by hiding irrelevant data.
2. Conceptual Layer
Represents the logical structure of the entire database.
Hides physical storage details from the user.
3. Internal Layer.
.Deals with how data is physically stored..
Focuses on indexing, compression, encryption, storage methods.
HI Query
Definition:.
A query is a request to retrieve or manipulate data in a database.
Example:
SELECT * FROM Employees WHERE salary > 50000;
Generalization and Specialization
Generalization (Bottom-up approach)
Combining two or more entities into a higher-level entity
Specialization (Top-down approach):
Dividing a higher-level entity into lower-level entities based on attributes.
Keys in DBMS
Primary Key: Uniquely identifies a record. (Cannot be NULL.)
.Candidate Key: All possible keys that can be primary keys
Super Key: Any combination of attributes that uniquely identifies a row.
Integrity Constraints
Entity Integrity: No NULL values in primary key..
Referential Integrity: Foreign key value must exist in referenced table.
Q Functional Dependencies (FD)
If A -> B, knowing A lets you know B.
Helps find redundancy and normalize data..
Example:
Student_ID > Student_Name.
= Closure of Functional Dependencies
The closure (F+) is the full set of dependencies that can be inferred from a given set.
How to find Closure:
Start with the attributes in the left-hand side.
Apply FDs repeatedly until no new attributes are added..
M Cover (Minimal Cover)
. A minimal set of FDs that preserve the original dependency set..
Should have:.
Only single attributes on the right-hand side.
o No redundant attributes on the left-hand side..
o No redundant dependencies..
3 Trigger
Trigger is an automatic action performed when a specific database event occurs
(INSERT, DELETE, UPDATE).
Example:
Auto-update the last_modified timestamp after updating a record..
j Normal Forms (1NF to 5NF)
1NF: Atomic values (no multiple values in a single field)
. 2NF: No partial dependency (full functional dependency)
3NF: No transitive dependency (A -> B -> C should be broken).-
BCNF: Each functional dependency A -> B, A should be a superkey
4NF: No multivalued dependency.
5NF: Lossless decomposition for join dependency.
G Decomposing and Synthesizing.
Decomposition: Breaking a relation into two or more relations
Synthesis: Combining smaller relations into bigger ones, ensuring dependency
preservation and lossless join.
y Multivalued Dependency (MVD)
When one attribute depends on another, independently of other attributes.
Example:
In a college database: a student can have multiple phone numbers and multiple courses.
Qj Multi-value vs Multivalued Dependency
Multi-value: A field having multiple values (bad for 1NF).
Multivalued Dependency (MvD): A constraint specifying multiple independent facts.
6NF (Sixth Normal Form)
Deals with temporal databases.
Decomposes tables to irreducible relations (no non-trivial join dependency).
I Transaction
A logical unit of database operations performed as a single unit
Properties (ACID):
Atomicity
Consistency
Isolation
Durability
Transaction Life Cycle
1. Active: Transaction is executing
2. Partially Committed: Last operation executed.
3. Failed: Transaction failed during execution.
4. Aborted: Changes rolled back
5. Committed: All operations successfully done
Q Types of Transactions
Flat Transaction: Simple, atomic.
Nested Transaction: Transactions within transactions.
Distributed Transaction: Across multiple databases.
Compensating Transaction: Undo a failed transaction logically.
5 Serial and Non-Serial Transactions
Serial: One transaction completes fully before the next starts
Non-Serial: Transactions overlap or run concurrently
Schedule
The order in which operations of concurrent transactions are executed.
Shadow Copy
Technique to keep backup copies of the database during transactions..
Allows easy rollback on failure.
Q Conflict and Non-conflict Transactions
Conflict: Operations that read/write the same data and affect each other.
. Non-Conflict: Operations that can be reordered safely.
If non-conflict -> we can SwAP operations for better performance.
eo View Serializability
. If two schedules produce the same final database state, they are view-serializable.
0 Lock and Types
Binary Lock: 0 (Unlocked) or 1 (Locked)
Shared Lock (S): Read-only access.
Exclusive Lock (X): Read and write access.
Q? Granting Locks
. Check Lock Compatibility Matrix to decide if a lock can be granted

Deadlock, Starvation, Two Phase Locking (2PL).
Deadlock: Circular waiting for resources.
Starvation: Some transactions wait forever.
Two Phase Locking:
oGrowing Phase (Acquiring Locks)
o Shrinking Phase (Releasing Locks)
4 Static Two Phase Locking
All locks acquired at once before execution starts..
Ensures deadlock prevention
M Lock Conversion
Upgrade: Shared -> Exclusive.
Downgrade: Exclusive -> Shared.
. File System Blocks
Unspanned Blocks: A record cannot cross block boundary..
. Spanned Blocks: A record can cross block boundaries
(3) File Organization
Unsorted Files: No order maintained..
Sorted Files: Ordered based on a key..
Master Table: Important table used for reference..
Ordered Field: Field based on which sorting happens..
Hashing: Use of hash functions to calculate location directly.
Overflow in Hashing.
Linear Probing: Place in next empty slot..
.Chaining: Create linked list of entries.
Extendible Hashing: Directory based hashing technique.
$ Indexing
An index is a data structure that improves the speed of data retrieval
Anchor Element in Indexing (Primary Index).
Insert, Delete, Create Index in B-Tree
Insertion: Insert key, split node if overflow.
Deletion: Remove key, merge or redistribute if underflow.
Creation: Build a balanced tree maintaining order of keys
M Rules for Query Tree to Optimized Query Tree
1. Apply selections as early as possible.
2. Perform projections as early as possible.
3. Combine cascading selections into a single selection.
4. Decompose complex selections into smaller ones
5. Perform joins on smaller tables first.
6. Push selections and projections down the query tree.
7. Reorder joins where applicable for efficiency.

Q1 SRTF vs SJF
Statement: "Shortest remaining time first (SRTF) behaves like shortest job first (SJF) if the set

Answer: True..

Q2 - CPU-bound vs I/O-bound and effect on context switches
Difference (simple):
CPU-bound process: does long CPU bursts, rarely issues I/O. It holds the CPU for long
stretches.
. I/O-bound process: does short CPU bursts and frequently does I/O (blocks often)
Effect on context switching:
I/O-bound processes cause more frequent context switches because they block for
I/O and the OS must switch to another process often. They produce many short CPU
bursts (more scheduler decisions)
before switching). If preempted by time quantum, they still cause switches but less
frequently than many small I/O bursts..
(So I/O-bound context-switch rate; CPU-bound context-switch frequency but each switch
may cost the same overhead.)

(dynamic priority) can schedule many task sets with U up to 100% (in theory) that RM
cannot.
Also when deadlines are dynamic or deadlines periods or tasks are not strictly
periodic, EDF's dynamic priorities give it an advantage.
patterns where EDF can meet deadlines and RM cannot), EDF will succeed where RM
fails.
Q5 - Can a system detect starvation? If yes how? If no, how to deal with
starvation?
Answer: Yes - detection and mitigation both possible.
Detection: The system can monitor per-process waiting times (time spent ready but not
scheduled) or number of times preempted. If a process's waiting time exceeds a configured
threshold, the system can treat that as starvation.
Mitigation (common technigues):
Aging: gradually increase the priority of processes that wait long, so they eventually get
CPU.
Fair schedulers: use fair-share / proportional-share schedulers (CFS in Linux style)
which bound starvation..
Admission / resource limits: limit how many high-priority tasks a user or class can
have so low-priority tasks aren't starved.
So detection is practical; the usual response is to apply aging or use a scheduler that
guarantees progress.

Interpretation (standard): you are given two black boxes implementing keyed functions
(F_1(k,m)) and (F_2(k,m)) (both accept a 64-bit key and 64-bit message and return a 64-bit
block). One of these black boxes implements DES(_k(m)) (the real DES encryption with key (k)
and message (m)); the other implements an arbitrary random function (f(k,m)). You can make
(chosen) queries ((k,m)\mapsto \text{output}) and want to decide which box is DES
Simple distinguishing idea (permutation test / birthday collision):
Property of DES: For any fixed key (k), the map (m\mapsto\text{DES}_k(m)) is a
permutation on 64-bit blocks (i.e. bijection - distinct plaintexts map to distinct
ciphertexts).
Random keyed function: For a truly random function (f(k,\cdot)) (for fixed (k)), the
mapping from plaintexts to outputs is a random function (not necessarily a permutation);
if you query enough different plaintexts under the same key, you will likely see collisions
among outputs.
Attack: For each candidate black box B:
1. Choose a fixed key (k_0)..
2. Query B(k_0, m) for many distinct plaintexts (m) (say collect (g) outputs).
3. Check whether any two outputs collide.
o If no collisions are observed (over a very large set of distinct plaintexts), that
suggests B is DES (since DES for fixed key is collision-free).
o If collisions are observed, B cannot be DES (since DES_k is injective).
Complexity & comments:
The birthday bound says collisions among 64-bit outputs appear after about (2^{32})
collisions with good probability against a random function.
Thus the approach is practical in principle but requires about (2^{32}) queries and
If the course question asks for "polynomial many queries" in a security parameter, note
(2^{32}) is exponential in the key-size parameter if key-size is treated as the security
parameter; however in practice the birthday attack is the standard distinguishing method.
Alternative notes (theoretical):.
Any keyed block cipher family (like DES) yields permutations for fixed key; a truly
random keyed function (mapping keyxmessage to output) behaves differently and can
be distinguished using collision statistics as above.
There also exist structural properties (e.g., DES complementation property or weak
keys) that could also be used to help distinguish with fewer queries if exploited cleverly.
but the generic and robust distinguishing test is the permutation/collision test described
above.

E(x_1)\opIus E(x_2)=A(x_1\opIus x_2)

Request (E(0)=b)
For each basis vector (e_i) (128 vectors) request (E(e_i)). Then

brute force (2^{128})
(If attacker only has a handful of known plaintexts but can choose plaintexts, the same idea with

\varphi(n) = n - (p+q) + 1 \quad\Rightarrow\quad p+q = n-\varphi(n)+1.

Q4 - DLP description, faster algorithm,
and ZK proof sketch (6)
Discrete Log Problem (DLP).
In a finite cyclic group (G=\langle g\rangle) of known order (m), given (g) and (h\in G), the DLP
is: find (x\in{0,\dots,m-1}) such that (g^x=h)..
Algorithm better than exhaustive search. (Two standard choices.)
Baby-step / Giant-step (Shanks). Let (m) be group order, set (m_0=\lceillsqrt m\rceil)
Precompute baby steps ({g^j:0\le j<m_0}) in a table. Then compute (g^{-m_0 i} h) for
(i=0,1,\dots) and Iook up matches. This finds (x=i m_0 + j) in time and memory (O(\sqrt
m)).
Pollard's Rho for Iogarithms. A randomised algorithm with expected time (O(\sqrt m))
and negligible memory. For multiplicative groups (large prime modulus) index-calculus
methods can be faster in practice and subexponential in the prime size..
Zero-Knowledge proof of knowledge (based on DLP hardness)..
A standard ZK protocol to prove knowledge of discrete log (x) for (h=g^x) is Schnorr's.
protocol (3-move, honest-verifier ZK):
1. Prover picks random (r), sends commitment (t=g^r)
2. Verifier sends random challenge (c) (small integer or from a large challenge space)..
3. Prover responds (s=r + c x \pmod m).
4. Verifier checks (g^s loverset{?}{=} t\cdot h^c)
Completeness, soundness (extractor from two transcripts with different challenges gives (x))
and zero-knowledge (a simulator can produce indistinguishable transcripts) are standard proofs
