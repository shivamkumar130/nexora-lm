1 Introduction
Quicksort is a highly efficient sorting algorithm based on partitioning an array into smaller sub-
arrays. Here’s a brief overview of how it works...
Quicksort is a highly efficient divide-and-conquer sorting algorithm. It works by selecting a pivot
element from the array, then partitioning the array into two subarrays: one with elements smaller
than or equal to the pivot and the other with elements greater than the pivot. The algorithm
recursively sorts the subarrays and, in the process, sorts the entire array.
1.1 Description of Quicksort
Quicksort is a divide-and-conquer algorithm, similar to merge sort, but more efficient in practice.
It uses the divide-and-conquer approach to sort arrays. Here’s a breakdown of the three steps
involved in the process for sorting a subarray A[p... r]:
A. Divide
In this step, the array is partitioned into two subarrays:
• Choose a pivot element, typically A[r], the last element in the array.
• Rearrange the elements so that:
– All elements in the subarray A[p... q− 1] are less than or equal to the pivot A[q].
– All elements in the subarray A[q + 1... r] are greater than the pivot.
By doing this, the pivot is in its correct sorted position. The index q is computed during the
partitioning process, which divides the array into two smaller parts.
Example:
For an array A = [9, 3, 7, 1, 8], selecting 8 as the pivot would lead to:
• Left subarray: [3, 7, 1]
• Right subarray: [9]
With the pivot 8 placed between them.
B. Conquer
After partitioning, the algorithm recursively sortsthe two subarrays:
• The subarray A[p... q− 1] (elements smaller than or equal to the pivot).
• The subarray A[q + 1... r] (elements larger than the pivot).
The recursion continues until the subarrays contain one or zero elements, at which point they are
already sorted.
C. Combine
In quicksort, no explicit combination step is required because the array is already sorted once the
recursive calls return. The pivot naturally divides the sorted parts, so the entire array A[p... r] is
now sorted.
2. Quicksort Implementation
QUICKSORT(A, p, r)
1 if p < r
2 q = PARTITION(A, p, r)
3 QUICKSORT(A, p, q - 1)
4 QUICKSORT(A, q + 1, r)
To sort an entire array A, the initial call is QUICKSORT(A, 1, A.length()).
2.1 Partitioning the Array
The key to the algorithm is the PARTITION procedure, which rearranges the subarray A[p... r] in
place.
PARTITION(A, p, r)
1 x = A[r] # Choose the pivot as the last element
2 i = p - 1 # i is the boundary of the "less than or equal to pivot" partition
3 for j in range(p, r):
4 if A[j] <= x:
5 i += 1
6 A[i], A[j] = A[j], A[i] # Swap elements to move them to the left partition
7 A[i + 1], A[r] = A[r], A[i + 1] # Place the pivot in its correct position
8 return i + 1 # Return the pivot index
The partition procedure is the key part of quicksort. It rearranges the elements of the subarray
A[p... r] such that elements smaller than the pivot are placed on one side and those larger on the
other. This is achieved by iterating through the array and swapping elements based on comparisons
with the pivot.
2.2 Loop Invariant Explanation
The PARTITION function works on an 8-element array. The function always selects a pivot element
x = A[r] (the last element of the subarray) to partition the array A[p... r]. During the process,
the array is split into four regions, which may or may not contain elements. These regions evolve
as the function progresses.
At the start of each iteration of thefor loop (lines 3–6), the regions maintain specific properties,
which we express as a loop invariant.
2.3 Loop Invariant
At the beginning of each iteration of the loop, for any array index k:
1. If p ≤ k ≤ i: - All elements in this range A[k] are less than or equal to the pivot x.
2. If i + 1 ≤ k ≤ j − 1: - All elements in this range A[k] are greater than the pivot x.
3. If k = r: - A[k] = x, meaning the pivot remains unchanged at position r.
These conditions ensure that the array is gradually being divided into two parts—elements
smaller than or equal to the pivot and elements larger than the pivot—while the pivot stays in
place at the end of the array. Once the loop finishes, the pivot is swapped into its correct position
between the two partitions.
3. PARTITION Procedure Explained
Here’s a step-by-step explanation of the PARTITION procedure, as applied to an 8-element array:
3.1 Initial Setup
- The last element of the arrayA[r] is selected as the pivot elementx. - Two pointers are maintained:
i (initially set to p − 1) and j (initially set to p).
3.2 Example Array:A = [2, 8, 7, 1, 3, 5, 6, 4], Pivot x = 4
1. (a) Initial state: - i = p − 1, j = p. - No elements have been compared yet, and none have
been assigned to either partition. - The pivot element x = 4 is not yet in place. - Initial
array: [2, 8, 7, 1, 3, 5, 6, 4].
2. (b) Compare A[j] = 2 with x = 4: - A[j] ≤ x, so: - Increment i: i = i + 1, now i = 0. - Swap
A[i] with A[j] (no actual swap since A[i] = A[j] = 2). - Array state: [2, 8, 7, 1, 3, 5, 6, 4]. -
Increment j.
3. (c) Compare A[j] = 8 with x = 4: - A[j] > x, so: - Do nothing; just increment j.
4. (d) Compare A[j] = 7 with x = 4: - Same as previous: A[j] > x.
5. (e) Compare A[j] = 1 with x = 4: - A[j] ≤ x: - Increment i to 1. - Swap A[i] with A[j]: -
Array state: [2, 1, 7, 8, 3, 5, 6, 4].
6. (f) Continue similarly until j reaches r.
7. (g) Finally, swap A[i + 1] with A[r]: - New array: [2, 1, 3, 4, 8, 5, 6, 7]. - Pivot is in the correct
position.
4. Time Complexity Analysis
The average case time complexity of Quicksort is O(n log n), where n is the number of elements in
the array. This is due to the logarithmic number of partitions and linear time for each partition.
However, in the worst case (when the smallest or largest element is always chosen as the pivot),
the time complexity degrades to O(n2). Randomly selecting a pivot or using the median-of-three
rule can help mitigate this risk and improve performance in practice.
5. Conclusion
Quicksort is an efficient and widely used sorting algorithm that performs well in average cases. It’s
essential to understand the mechanics of the partitioning process and how it contributes to the
algorithm’s performance.
