### **Graph Representation**
Graphs are fundamental in computer science, and choosing an appropriate representation
directly impacts algorithm efficiency. Two standard methods to represent a graph \(G = (V, E) I)
are the **adjacency list** and the **adjacency matrix**, each suited for specific scenarios.
##* **1. Adjacency List**
- An adjacency list represents the graph as an array of lists, where each list corresponds to a.
vertex and contains all its adjacent vertices.
- For **directed graphs**, each edge \(u, v)\) appears in \(u I)'s list, making the total memory
usage proportional to \(IV] + [EI \).
- For **undirected graphs**, each edge \((u, v)\) appears in both I(u I)'s and I(v \)'s lists,
requiring \(2IEI \) space.
- **Advantages**:
- Supports weighted graphs by storing edge weights alongside vertices.
- **Limitations**:
- Checking for the presence of an edge \((u, v)\) requires traversing \(u I)'s list, which can take
\(\mathcal{O}(\text{degree}(u)) l) time.

Breadth-First Search (BFS)
Breadth-First Search (BFS) is a fundamental graph traversal algorithm that explores the vertices
of a graph layer by layer. It is widely used in various applications, such as finding the shortest
path in an unweighted graph, determining connectivity, and building spanning trees.

Given a graph G=(V,E)G = (V, E)G=(V,E) and a distinguished source vertex sss, BFS
